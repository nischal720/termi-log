# Contributing to Termilog

First off, thank you for considering contributing to Termilog! It's people like you that make Termilog such a great tool.

## Code of Conduct

Help us keep Termilog open and inclusive. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs

- **Search for existing issues** before creating a new one.
- **Provide a clear and descriptive title**.
- **Describe the exact steps to reproduce the problem**.
- **Include environment details** (Node version, OS, etc.).

### Suggesting Enhancements

- **Check if the feature is already requested**.
- **Explain why this enhancement would be useful**.

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. **If you've added code that should be tested, add tests**.
3. **Ensure the test suite passes**.
4. **Make sure your code lints**.
5. **Issue that pull request!**

## Development Setup

1. Clone your fork.
2. Run `npm install` to install dependencies.
3. run `npm test` to verify the current state.

## Style Guide

- Use 2 spaces for indentation.
- Follow existing patterns in the codebase.

Thank you for your contributions!
